Работы

Экосистема разработки программ с открытым кодом

Целью данной лабораторной работы является создание собственной персональной страницы-отчёта в github

>> <

  • Шаг 1. Зайти в репозиторий stankin/inet-2017 и создать собственный форк.

  • Шаг 2. Клонировать репозиторий на локальный компьютер. Для этого необходимо использовать утилиту git bash.

  • Шаг 3. В программе git bash ввести команду git clone, указав скопированную ссылку в качестве аргумента.

  • Шаг 4. Создать файл индивидуальной страницы. Для этого необходимо создать папку, названную в виде "Фамилия_И_О". Внутри данной папки необходимо создать файл index.html, заполнив его стандартной структурой html файла.

  • Шаг 5. В программе git bash выполнить команду git add для добавления файлов под версионный контроль. После этого необходимо выполнить команду git commit -m "", дописав в кавычках информацию о коммите.

  • Шаг 6. Для публикации коммита в основном репозитории необходимо выполнить команду git push origin master.

  • Шаг 7. Затем необходимо предложить коммит для внесения в основной проект. Для этого в репозитории GitHub необходимо нажать кнопку Pull Request.

Описание

Приложение решает краевую задачу для уравнения теплопроводности в одномерном пространстве.

Требования к программе

  1. Создать интерактивную форму для отображения приложения.
  2. На форму добавить поле "Начальные условия", в дальнейшем здесь будут вводиться формулы начальных условий для всех точек.
  3. Создать два поля ввода формул краевых точек (аналогично предыдущему пункту).
  4. Добавить поле для ввода формулы, которая вносит условия для изменения температуры на точках.
  5. Создать отдельный блок для ввода следующих данных: длина, наблюдаемое время, делители длины и времени
  6. Добавить кнопку "Рассчитать"
  7. По окончании вычислений строится трёхмерный температурный график

После добавления на форму двух компьютеров и коммутатора, а так же их настройки, необходимо проверить отправление пакетов с устройства на устройство.

При добавлении на форму ещё двух компьютеров, настроенных для подсети B, подключенных к общему коммутатору, необходимо проверить работоспособность сети. В данном варианте пакеты между устройствами, находящимися в разных сетях ходить не будут, т.к. коммутаторы находятся на 2 уровне модели OSI и не умеют обрабатывать IP-запросы

После того как на форму добавлен маршрутизатор и произведена его настройка, необходимо отправить пакеты из одной сети в другую.

Так как у компьютеров не указан шлюз по умолчанию, то устройство, при направлении пакета, предназначенного устройству из другой сети, не знает кому передавать свой запрос.

Для того, чтобы между устройствами из разных сетей ходили пакеты, необходимо прописать адреса интерфейсов роутера как шлюзы по умолчанию.

После добавления, подключения и настройки компьютера, коммутатора и маршрутизатора, необходимо проверить возможность передачи данных между сетями. Так как маршрутизатор 1 не знает о сетях маршрутизатора 2, а маршрутизатор 2 не знает о сетях маршрутизатора 1, пакеты между этими маршрутизаторами не ходят.

Для передачи между маршрутизаторами данных необходимо прописать статическую маршрутизацию сетей между этими двумя маршрутизаторами.

Экзаменационный вопрос

Выберите одно верное утверждение.

Рефакторинг – это:

  1. Процесс изменения внутренней структуры программы, на затрагивающий ее внешнего поведения и имеющий целью облегчить понимание ее работы, устранить дублирование кода, облегчить внесение изменений в ближайшем будущем.
  2. Имитатор, задача которого состоит в имитации управления каким-либо процессом, аппаратом или транспортным средством.
  3. Комплекс программных, аппаратных средств или их сочетание, предназначенное для копирования функций одной вычислительной системы на другую, отличной от первой, вычислительную систему, таким образом, чтобы скопированное поведение как можно ближе соответствовало поведению оригинальной системы.
